P0 Critical — Fix Now
⚡
Fix mobile LCP on all 6 pages (currently 9.4 s – 27.2 s; threshold: 2.5 s)
Owner: Dev + SEO Deadline: 15 May 2026 Mobile ranking risk Carryover from March audit
  1. Identify LCP element on each page via Chrome DevTools or WebPageTest. On homepage it is the hero image/banner above the fold.
  2. Add <link rel="preload" as="image"> for the LCP element in the <head> of each page template.
  3. Convert all above-fold images to WebP (or AVIF). Serve via CDN with appropriate cache-control headers.
  4. Set loading="lazy" on all below-fold images. Remove it from above-fold LCP images.
  5. Inline critical CSS. Defer render-blocking stylesheets.
  6. Re-run PageSpeed Insights on each page after deploy and validate LCP <2.5 s.
📈 Expected impact: Improved mobile ranking signals across all 6 pages. Google Search Console CWV report should move from "Poor" to "Needs Improvement" or "Good" within 28 days of fix.

Fix homepage desktop TBT = 1,170 ms (threshold: <200 ms good)
Owner: Dev Deadline: 20 May 2026 Desktop interactivity blocked
  1. Open Chrome DevTools → Performance tab → record homepage load. Identify long tasks blocking main thread (>50 ms).
  2. Audit Tag Manager: remove or defer analytics tags, heatmap tools, and chat widgets that fire on page load.
  3. Split large JavaScript bundles using dynamic imports or code splitting (webpack/Next.js).
  4. Move non-critical scripts to defer or async. Remove any synchronous scripts above the fold.
  5. Target: TBT <200 ms (desktop), Performance score >80 (currently 52).
📈 Expected impact: Homepage desktop score improves from 52 → 80+. Better UX for all desktop users. May impact homepage ranking quality score.
