P0 — Breadcrumb Schema "Unnamed item" Bug (Sitewide)

ALL pages return breadcrumb schema with the name property missing or empty. Google cannot render rich breadcrumb results without names. This is almost certainly a WordPress plugin misconfiguration — check Yoast SEO or RankMath breadcrumb settings. One fix = sitewide improvement across all pages.

🔴

P0 — Exchange Rates Page Missing From Sitemap

The youtrip-exchange-rates page is the site's #3 traffic page (6,972 clicks). GSC URL Inspection returned no sitemap reference for this URL — all other inspected pages return a sitemap URL. Verify manually in GSC → Sitemaps. If missing, add it to post-sitemap2.xml or force a sitemap regeneration.

🔴

P0 — Stale 2020 PDF Ranking in Search Results

/wp-content/uploads/sites/2/2020/09/YouTripPerks-2020-Programme_Applicable-MCC-Codes.pdf is appearing in search results and generating 187 clicks. This is a 2020 document — the MCC codes and programme terms are likely outdated. Options: (1) Add X-Robots-Tag: noindex via server config, or (2) set up a redirect to the current perks page. Do not leave stale content ranking.

🟡

Warning — Mobile Usability Unconfirmed via API

All inspected pages returned MobileUsabilityResult = VERDICT_UNSPECIFIED. The API could not confirm mobile usability status. This is not necessarily a problem — it may be an API limitation. Check GSC → Core Web Vitals → Mobile directly to rule out mobile usability issues.

🟡

Warning — Best Savings Accounts Page Severely Underlinked

The best-savings-accounts-singapore page has only ONE internal link pointing to it — from an irrelevant Okinawa travel guide. This is a competitive finance keyword (est. 12,000 vol/mo) sitting at pos 13.5. Internal link equity from relevant finance pages could meaningfully help rankings. See Actions → P2 for the fix plan.

Critical Sitewide Bug — Breadcrumb "Unnamed item" on All Pages

Every inspected page returns breadcrumb schema where the name property is missing or empty. Google's Rich Results Test will show "Unnamed item" entries. Rich breadcrumbs will not render in SERPs until this is fixed. Check the WordPress breadcrumb plugin (Yoast or RankMath) and confirm the name field is being passed in the JSON-LD output. This is a single fix with sitewide impact.
