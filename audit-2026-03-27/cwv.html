🔍 Key Findings
🚨

Homepage LCP: 25.3s Mobile — Catastrophic

The homepage is 10× over the "poor" threshold. This is almost certainly caused by a render-blocking hero image or above-the-fold asset that isn't preloaded on mobile. Desktop LCP is 3.0s — a >8x gap between mobile and desktop on the same page indicates the issue is mobile network + image size, not server speed. Fix: identify the LCP element via Chrome DevTools on a throttled mobile connection, then set fetchpriority="high" and serve WebP at <100KB.

🚨

About YouTrip: 1,120ms TBT Mobile — Heavy JS Blocking

Total Blocking Time of 1,120ms on mobile means the main thread is locked for over a second during load. Combined with a perf score of 41, this page has significant JS overhead. Likely causes: multiple analytics scripts (GTM, Pixel, GA4), a heavy page builder, or third-party embeds loading synchronously. Fix: defer non-critical JS; audit active plugins on this page template.

⚠️

All Article Pages: LCP ~6s Mobile — Consistent Pattern

SGD to JPY, Travel Insurance, Exchange Rates, and SG Public Holidays all cluster around 5.9–6.6s LCP on mobile. The consistency suggests a sitewide template issue — likely the hero/header image on the article template isn't preloaded. Fixing the template would improve all 4 pages simultaneously. Fix: add LCP image preload hint to the article page template header.

✅

CLS is Clean Everywhere — Not an Issue

All 6 pages return CLS <0.025 on both mobile and desktop. Layout stability is not a problem. This can be deprioritised — focus effort entirely on LCP and TBT.
